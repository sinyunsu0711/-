1. 연구 주제

Mapbox GL JS와 Three.js를 활용한 실시간 글로벌 재난 및 주요 사건의 3차원 시각화 시스템 연구 및 구현

2. 서론

가. 연구 동기 및 필요성

기존의 지리 공간 정보 시각화는 주로 2차원(2D) 지도 마커나 폴리곤 형태로 데이터를 표현하는 데 집중되어 있어, 데이터의 규모나 중요도 같은 속성 값을 사용자에게 직관적이고 효과적으로 전달하는 데 한계가 있었습니다. 특히 지진, 산불, 허리케인 등 영향도의 차이가 큰 글로벌 사건들을 단일 평면 위에 나열할 경우, 정보의 중요성이 제대로 부각되지 않을 수 있습니다.

본 연구는 Mapbox의 지도 렌더링 환경 위에 Three.js(WebGL)를 통합하여 데이터를 3차원 모델(Box Geometry)로 표현함으로써, 사건의 영향도를 모델의 높이나 색상으로 시각적으로 인코딩하는 혁신적인 방법을 제시합니다. 이는 사용자에게 공간적, 규모적 측면에서 정보를 입체적으로 전달하여 **데이터 이해도와 몰입도를 극대화할 필요성**에서 출발했습니다.

나. 연구 목적

  • **안정적인 3D 렌더링 환경 구축**: Mapbox GL JS의 Custom Layer 기능을 활용하여 Three.js 씬을 지도 레이어와 정확히 동기화하고, 3D 모델을 안정적으로 렌더링하는 환경을 구축한다.
  • **데이터 속성 기반 3D 시각화 구현**: Mock 데이터를 기반으로 사건 유형(재난, 정상회담 등) 및 영향도에 따라 모델의 높이와 색상이 자동으로 변화하는 맞춤형 3D 프리즘 모델을 지도 위에 배치한다.
  • **실시간 상호작용 시스템 개발**: 3D 모델 클릭 시 Three.js의 Raycasting 기술을 사용하여 객체를 정확히 감지하고, 해당 사건의 상세 정보를 메시지 박스로 출력하는 상호작용 인터페이스를 제공한다.

다. 연구 문제

  • **좌표계 변환 문제**: Mapbox의 지리 좌표계(위도/경도)를 Three.js의 3D 월드 좌표계(Mercator Coordinate)로 어떻게 정확하게 변환하여 3D 모델을 지도상에 오차 없이 배치할 수 있는가?
  • **렌더링 동기화 및 성능 문제**: 지도 뷰포트(줌, 회전, 기울기) 변경 시 Three.js 씬을 Mapbox 렌더링 사이클과 동기화하여 끊김 없는 동적 3D 시각화를 유지할 수 있는가?
  • **보안 환경 대응**: iFrame 등 제한적인 웹 환경에서 Mapbox 초기화 시 발생하는 크로스-오리진(Cross-Origin) 보안 오류를 효과적으로 예외 처리하고 사용자 UI의 안정성을 확보할 수 있는가?

3. 이론적 배경

가. 관련 이론/기술 및 선행 연구 조사

분류 이론/기술 설명 및 적용 방안
지도 플랫폼 Mapbox GL JS WebGL을 기반으로 벡터 타일과 사용자 정의 스타일을 렌더링하는 핵심 플랫폼. **Custom Layer** 기능을 사용하여 외부 WebGL(Three.js) 렌더링을 지도에 통합하는 기반을 제공함.
3D 렌더링 Three.js 및 WebGL 웹 브라우저에서 3차원 그래픽 및 상호작용을 구현하는 JavaScript 라이브러리. **THREE.BoxGeometry**를 사용하여 사건의 영향도를 시각화하는 3D 프리즘 모델을 생성하고 광원(Light)을 설정하는 데 사용됨.
좌표 변환 Mapbox Mercator Coordinate 위도/경도 좌표를 Mapbox의 3D 월드 공간에 맞게 변환하는 클래스. **mapboxgl.MercatorCoordinate.fromLngLat**을 사용하여 3D 모델을 정확한 지리적 위치에 배치하는 데 필수적임.
상호작용 Raycasting 3D 씬에서 마우스 포인터의 2D 화면 좌표를 3D 공간의 광선으로 변환하여, 이 광선과 충돌하는 객체(3D 모델)를 감지하는 기술. **3D 객체 클릭 이벤트 처리**에 활용됨.

4. 연구 내용 및 방법

가. 연구 대상, 절차, 방법 및 일정

* 연구 대상: **가상의 글로벌 주요 사건 데이터셋 (MOCK_EVENTS)**. 각 데이터는 id, title, type, latitude, longitude, impact, country, model_color, model_height 속성을 포함한다.

* 방법: HTML, Tailwind CSS, JavaScript를 사용한 **프로토타입 구현 및 기능 검증 방식**.

단계 연구 절차 및 내용 주요 구현 기술
1단계 기반 환경 설정 및 데이터 정의 Mapbox GL JS, Three.js, Mock Data, Tailwind CSS
2단계 3D 렌더링 통합 (Custom Layer) Mapbox Custom Layer, THREE.Camera, THREE.Scene, THREE.WebGLRenderer
3단계 3D 모델 배치 및 시각화 MOCK_EVENTS 반복 처리, THREE.BoxGeometry, Mercator Coordinate 변환, 높이/색상 인코딩
4단계 상호작용 및 디버깅 Raycasting을 이용한 클릭 이벤트 처리, 상세 정보 메시지 출력, 지도 초기화 보안 오류 예외 처리
